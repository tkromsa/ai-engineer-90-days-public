# Weekly system — AI engineer 90 days

## Cíl týdne
Každej týden musí mít:
- 1 konkrétní technickej cíl
- 1 dokončenej artefakt
- 1 public-facing update nebo commit trail

## Pondělí
- vyber scope týdne
- napiš 3 deliverables max
- připrav branch / repo / issue list

## Úterý
- hlavní build day
- bez zbytečnýho doomscrollu po repách

## Středa
- debug + tests + cleanup

## Čtvrtek
- docs day
- README, architecture note, examples, screenshots

## Pátek
- ship day
- push, release note, changelog, demo capture

## Sobota
- review:
  - co fungovalo
  - co byl bottleneck
  - co bylo PNJ

## Neděle
- lehký plán dalšího týdne
- žádnej velkej context switch

## Týdenní checkpoint otázky
- Co jsem reálně postavil?
- Co je na GitHubu vidět?
- Co umím dnes vysvětlit líp než minulý týden?
- Kterej problém jsem fakt vyřešil, ne jen obešel?
- Co bych ukázal někomu cizímu za 3 minuty?

## Anti-bullshit guardrails
- Když není artefakt, není hotovo
- Když není README, není to portfolio
- Když není demo nebo output, je to jen záměr
- Když tomu nerozumíš, nepřikrášluj to
